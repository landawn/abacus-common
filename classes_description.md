| N<br>CommonUtil | Primary null-safe static facade for the Abacus library exposing 1000+ utility methods for primitive, Object, String, Array, Collection, Map and JavaBean operations. `N` is a thin extension of its sealed parent **CommonUtil** (which holds the core implementations); together they provide null/empty checks (isEmpty, isNullOrEmpty, notBlank, defaultIfNull), equality and hashing (equals, deepEquals, hashCode), array/collection mutation by index vs value (add, addAll, removeAt, removeRange, remove, removeAll, deleteAll, replace, fill, shuffle, rotate, reverse, swap), searching and stats (frequency, frequencyMap, contains, indexOf, lastIndexOf, distinct, min, max, median, average, sum, percentilesOfSorted), set algebra (intersection, union, difference, symmetricDifference, disjoint, partition), grouping/conversion (groupBy, toMap, toList, asList, asSet), sorting and binary search (sort, parallelSort, binarySearch), functional iteration with checked-exception support (forEach, map, filter, reduce, anyMatch, allMatch, noneMatch), bean and type conversion (copy, merge, convert, valueOf, toJson, toXml), async/threading helpers (sleep, sleepUninterruptibly, asyncExecute, runUninterruptibly), and stream/IO bridges. | Numbers | Comprehensive static utility facade for numerical operations across all primitive numeric types and their wrappers. Also exposes commonly used numeric constants (zero, one, minus-one) for each primitive wrapper type. All methods are static, thread-safe, null-safe, and performance-optimized. |
| Strings<br>ExtractStrategy<br>StrUtil | Comprehensive static utility facade for string operations: validation (isEmpty, isBlank, isValidEmailAddress, isJavaIdentifier, isBase64), transformation (reverse, abbreviate, capitalize, uncapitalize, swapCase), case conversion (upperCase, lowerCase, toCamelCase, toSnakeCase), padding (padStart, padEnd, center), trimming (trim, strip, trimToNull), searching (indexOf, contains, startsWith, endsWith), replacement (replace, replaceAll, replaceFirst with regex support), comparison with locale awareness, safe parsing (parseBoolean, parseInt, parseDouble), encoding (Base64, URL, character-set conversion), joining/splitting with delimiter or pattern, and extraction utilities (extractNumbers, extractEmails, substringsBetween). Uses `offset`/`count`/`len` parameter conventions, defaults to `Charsets.DEFAULT`, wraps checked `IOException` in `UncheckedIOException`, manages buffers/resources automatically, and is null-safe and thread-safe. |
| Dates<br>DateTimeFormat<br>DayOfWeek<br>ISO8601Util | Comprehensive static facade for date/time manipulation spanning legacy types (`java.util.Date`, `Calendar`, `GregorianCalendar`, SQL `Date`/`Time`/`Timestamp`, `XMLGregorianCalendar`) and modern `java.time` types. The nested **DateTimeFormat** enum (LONG, ISO_8601_DATE_TIME, ISO_8601_TIMESTAMP) selects predefined serialization formats; the nested **DayOfWeek** enum maps SUNDAY–SATURDAY to integers 0–6 via `intValue()` and `valueOf(int)`; the package-private **ISO8601Util** helper provides fast, GC-friendly ISO-8601 parse/format routines adapted from Jackson. |
| Beans | Static utility facade for JavaBean introspection, property access, transformation, and lifecycle management built on cached reflection metadata. | Array | Static utility facade for array creation and primitive/wrapper array conversion across 1D, 2D, and 3D shapes. Uses half-open `fromIndex`/`toIndex` ranges, returns `null` for `null` inputs rather than throwing, and is null-safe and thread-safe. |
| Iterables | Static utility facade for aggregation, search, set-algebra, and combinatorial operations over `Iterable`/`Iterator`/array inputs, complementing `N` with Optional/Nullable-returning variants. Capabilities include min/max and ordering queries (min, max, minBy, maxBy, minMax, median, kthLargest returning `Nullable`/`Optional`/primitive optionals), numeric aggregation across width-preserving types (sumInt, sumIntToLong, sumLong, sumDouble, sumBigInteger, sumBigDecimal, averageInt, averageLong, averageDouble, averageBigInteger, averageBigDecimal), index/search (indexOf, lastIndexOf, findFirstOrLast, findFirstAndLast, findFirstOrLastIndex, findFirstAndLastIndex), lazy unmodifiable set views (union, intersection, difference, symmetricDifference, subSet, powerSet), combinatorics (rollup, permutations, orderedPermutations, cartesianProduct), bulk mutation helpers (fill, copyInto, copyRange, asReversed), and null-selection helpers (firstNonNull, lastNonNull). | Iterators | Static utility facade for creating, transforming, and consuming `Iterator` instances with lazy, memory-efficient semantics. Capabilities include positional access (elementAt returning `Nullable`), search and counting (indexOf, frequency, occurrencesOf, count with predicates), transformation (map, flatMap, flatmap, filter, distinct, distinctBy, skipNulls), slicing (skip, limit, skipAndLimit, takeWhile, dropWhile, skipUntil), repetition (repeat, repeatElements, repeatCollection, cycle, cycleToSize), composition (concat, merge, mergeSorted, zip, unzip), iterator generation (generate, generateInfinite), bulk consumption (forEach with single- or multi-threaded reader/processor pools via `ExecutorService`, forEachNonNull), and stream/`ObjIterator` bridges. All methods are static, null-safe, lazy where possible, and thread-safe at the method-invocation level; many return `Nullable` to disambiguate absent vs `null` elements. |
| Maps | Static utility facade for Map creation, querying, transformation, and restructuring with Optional/Nullable-based value retrieval. Capabilities include map construction from parallel collections (zip), five families of value retrieval — existence-aware (getIfExists returning `Nullable`), null-as-absent (getOrDefaultIfAbsent, getOrEmptyListIfAbsent, getOrEmptySetIfAbsent, getOrEmptyMapIfAbsent), get-or-create (getOrPutIfAbsent, getOrPutListIfAbsent), type-converting (getAsInt, getAsLong, getAsBoolean, getAsString, getAsLocalDate, getAsOptional, getAsOrDefault), and path-based deep navigation (getByPath, getByPathIfExists, getByPathOrDefault using dot-and-`[index]` syntax) — plus entry-pair containment (containsEntry), filtering and inversion (filter, filterByKey, filterByValue, invert, flatInvert), key rewriting (replaceKeys), set-style comparison (difference, symmetricDifference, intersection), conditional mutation (putIfAbsent, putAllIf, removeIf, removeIfKey, removeIfValue, replace, replaceAll, merge), view extraction (keySet, values, entrySet), and nested-map flattening/unflattening (flatten, unflatten, flatToMap). All methods are static, null-tolerant for the map argument (validating non-null defaults and predicates), and thread-safe. |
| Index | Static utility facade for index-of searches across primitive arrays, Object arrays, `Collection`/`List`, `Iterator`, and `String`/`CharSequence`, returning `OptionalInt` rather than the magic `-1`. Method families include first-occurrence (of), last-occurrence (last), subarray/sublist matching (ofSubArray, lastOfSubArray, ofSubList, lastOfSubList), case-insensitive string search (ofIgnoreCase, lastOfIgnoreCase), all-match indices returning a `BitSet` (allOf with element or predicate), and range-bounded variants accepting a start index. All methods are static, stateless, null-safe, and thread-safe. |
| PrimitiveList&nbsp;(abstract)<br>BooleanList<br>ByteList<br>CharList<br>ShortList<br>IntList<br>LongList<br>FloatList<br>DoubleList | Zero-boxing, primitive-specialized counterparts of `ArrayList<Wrapper>` backed by compact primitive arrays with `RandomAccess` semantics. **PrimitiveList** is the abstract template (parameterized on boxed-type `B`, primitive-array type `A`, concrete-list type `L`) defining the common contract: storage management, capacity growth, serialization, collection interop (`toArray`, `boxed`, `toList`, `toSet`, `toCollection`), and iteration. The eight concrete subclasses (**BooleanList**, **ByteList**, **CharList**, **ShortList**, **IntList**, **LongList**, **FloatList**, **DoubleList**) provide type-specialized operations: indexed access (`get`, `set`, `add`, `addAll`, `remove`, `removeAll`, `removeIf`, `removeAllOccurrences`, `removeDuplicates`, `retainAll`, `clear`), search/sort (`indexOf`, `lastIndexOf`, `contains`, `containsAll`, `containsAny`, `binarySearch`, `sort`, `parallelSort`, `reverseSort`, `reverse`, `rotate`, `shuffle`, `swap`), transformation (`replaceAll`, `replaceIf`, `fill`, `copy`, `split`, `forEach`, `forEachIndexed`), set ops (`intersection`, `difference`, `symmetricDifference`, `distinct`), numeric/statistical aggregates on numeric variants (`min`, `max`, `median`, `sum`, `average`, `kthLargest`), functional pipelines (`filter`, `map*`, `stream`, `iterator`), and range factories (`range`, `repeat`, `random`). Memory-efficient (no `Integer`/`Double` wrappers), but not thread-safe — external synchronization required for concurrent use. |
| BaseStream&nbsp;(interface)<br>Stream&lt;T&gt;<br>IntStream<br>LongStream<br>FloatStream<br>DoubleStream<br>CharStream<br>ByteStream<br>ShortStream | Enhanced, lazy, functional pipeline API extending Java Streams with richer operators, primitive specializations, and fine-grained parallel control. **BaseStream** is the root interface defining lifecycle (sequential/parallel mode switching via `sequential()`/`parallel(maxThreadNum, splitor, executor)`, `close()`, `onClose()`, `iterator()`, `splitor`, `asyncRun`/`asyncCall`), and the lazy intermediate vs. terminal operation contract (annotated with `@IntermediateOp`/`@TerminalOp`/`@LazyEvaluation`/`@ParallelSupported`/`@SequentialOnly`). **Stream&lt;T&gt;** is the generic object stream with intermediate ops (`filter`, `filter(predicate, dropAction)`, `map`, `mapIfNotNull`, `flatMap`/`flatmap`/`flattMap`, `mapMulti`, `mapPartial`, `select`, `peek`, `skip`, `limit`, `step`, `distinct`/`distinctBy`, `sorted`/`sortedBy`/`reverseSorted`, `takeWhile`, `dropWhile`, `skipUntil`, `prepend`/`append`, `intersection`/`difference`/`symmetricDifference`, `cycled`, `rotated`, `shuffled`, `indexed`, `pairWith`, `slidingMap`/`sliding`/`window`/`chunked`/`split`, `groupBy`/`groupByToEntry`, `partitionBy`, `top`, `cached`, `buffered`, `delay`/`rateLimited`) and terminals (`forEach`, `forEachIndexed`, `forEachUntil`, `toList`/`toSet`/`toArray`/`toMap`/`toMultimap`/`toMultiset`/`toDataset`/`toImmutableList`, `reduce`, `collect`, `count`, `min`/`max`/`first`/`last`/`onlyOne`/`elementAt`/`kthLargest`, `anyMatch`/`allMatch`/`noneMatch`/`nMatch`, `findFirst`/`findAny`/`findLast`, `join`/`println`, `persist`/`persistToCSV`/`persistToJSON`). The seven primitive variants — **IntStream**, **LongStream**, **FloatStream**, **DoubleStream**, **CharStream**, **ByteStream**, **ShortStream** — mirror those operations on unboxed values with arithmetic terminals (`sum`, `average`, `summarize`/`summaryStatistics`, `kthLargest`, range factories like `range(start,end)`/`rangeClosed`) and cross-stream conversions (`boxed`, `asLongStream`, `asDoubleStream`, `mapToObj`, `mapToInt`). Pipelines are single-use (closed after a terminal op), auto-closeable, lazy, and switchable between sequential and parallel modes (`parallel(maxThreadNum, executor)`); sequential pipelines are not thread-safe but operations are stateless-by-default for safe parallel execution. |
| EntryStream&lt;K,V&gt;<br>Seq&lt;T,E&gt; | Two specialized stream-like pipelines complementing the main `Stream` API. **EntryStream&lt;K,V&gt;** is a `final` stream of `Map.Entry<K,V>` (extends `StreamBase`) tailored for key-value processing: construction from maps (`of(Map)`, `concat(maps...)`), key/value transforms (`mapKey`, `mapValue`, `mapKeyPartial`, `mapValuePartial`, `flatMapKey`, `flatMapValue`, `inversed`, `selectByKey`/`selectByValue`, `filterByKey`/`filterByValue`), aggregation (`groupBy`, `groupByToEntry`, `partitionBy`, `collectByKey`, `toMap`/`toImmutableMap`/`toMultimap`/`toMultiset`), key/value extraction (`keys`, `values`, `entries`), and standard pipeline operations (`filter`, `map`, `sorted`/`sortedByKey`/`sortedByValue`, `distinctByKey`/`distinctByValue`, `skip`, `limit`, `forEach`, `reduce`, `count`). Supports parallel mode. **Seq&lt;T, E extends Exception&gt;** is a sequential-only, lazy sequence that propagates checked exceptions through the pipeline (parameterized on element type `T` and declared exception `E`), making it ideal for I/O, JDBC, and reflection-heavy work without exception wrapping; provides the same shape of intermediate/terminal operators as `Stream` (`filter`, `map`, `flatMap`, `sorted`, `distinct`, `skip`, `limit`, `forEach`, `reduce`, `collect`, `toList`/`toSet`/`toMap`, `groupBy`, `partitionBy`), I/O-aware factories (`Seq.ofLines(Path)`, `Seq.of(Iterable)`), and a `stream()` bridge into the unchecked `Stream` world. Both are single-use, auto-closeable, and not thread-safe across threads. |
| Collectors | Sealed abstract factory class (extends `java.util.stream.Collectors` conceptually) producing `java.util.stream.Collector` instances for use with both JDK and abacus `Stream`/`EntryStream` pipelines. Provides standard JDK collectors plus rich abacus-specific extensions: collection builders (`toList`, `toUnmodifiableList`, `toImmutableList`, `toSet`, `toImmutableSet`, `toQueue`, `toDeque`, `toCollection(Supplier)`, `toMultiset`, `toLongMultiset`), map/multimap collectors (`toMap`, `toMap(merge)`, `toImmutableMap`, `toLinkedHashMap`, `toConcurrentMap`, `toBiMap`, `toMultimap`, `toListMultimap`, `toSetMultimap`), primitive list collectors (`toBooleanList`, `toByteList`, `toCharList`, `toShortList`, `toIntList`, `toLongList`, `toFloatList`, `toDoubleList`), grouping/partitioning (`groupingBy`, `groupingByConcurrent`, `partitioningBy`, with downstream collector overloads), counting/aggregation (`counting`, `countingToInt`, `countingToLong`, `summingInt`/`summingLong`/`summingDouble`/`summingBigInteger`/`summingBigDecimal`, `averagingInt`/`averagingLong`/`averagingDouble`/`averagingBigInteger`/`averagingBigDecimal`), reduction (`reducing(BinaryOperator)`, `reducing(identity, op)`, `reducing(identity, mapper, op)`), summary statistics (`summarizingInt`, `summarizingLong`, `summarizingDouble`, `summarizingBigInteger`, `summarizingBigDecimal`, plus `Char`/`Byte`/`Short`/`Float` variants), min/max (`minBy`, `maxBy`, `minOrElseThrow`, `maxOrElseThrow`, `minMax`), string assembly (`joining`, `joining(delimiter, prefix, suffix)`), composition combinators (`mapping`, `flatMapping`, `filtering`, `collectingAndThen`, `teeing(c1,c2,merger)`, `combine`), and first/last selectors (`first`, `last`, `onlyOne`). All collectors are stateless factory methods returning fresh `Collector` instances; the resulting collectors are safe for concurrent stream use only when the underlying container/strategy is documented as concurrent (e.g., `groupingByConcurrent`, `toConcurrentMap`). |
| StreamBase<br>AbstractStream<br>ArrayStream / IteratorStream<br>ParallelArrayStream / ParallelIteratorStream<br>ArrayIntStream / IteratorIntStream / ParallelArrayIntStream / ParallelIteratorIntStream<br>(and Long/Double/Float/Char/Byte/Short equivalents)<br>ArrayEntryStream / IteratorEntryStream / ParallelIteratorEntryStream | Stream implementation classes (package-private internals — not part of the public API). **StreamBase** is the shared abstract superclass holding close-handler chaining, async helpers, and parallel-mode plumbing. **AbstractStream&lt;T&gt;** implements the bulk of `Stream<T>` operator logic atop an iterator. The concrete `ArrayXxxStream` variants wrap a backing array slice (offset/length) for direct random access; `IteratorXxxStream` variants wrap an `Iterator`/primitive-iterator for lazy/unknown-size sources; `ParallelArrayXxxStream` and `ParallelIteratorXxxStream` add fork/executor-driven parallel evaluation (controlled by `maxThreadNum`, `splitor`, and an `AsyncExecutor`). Analogous triples exist for every primitive type (`Int`, `Long`, `Double`, `Float`, `Char`, `Byte`, `Short`) and for `EntryStream` (`ArrayEntryStream`, `IteratorEntryStream`, `ParallelIteratorEntryStream`). Callers obtain instances only via factory methods on `Stream`/`IntStream`/`EntryStream`/etc. Sequential variants are not thread-safe; parallel variants coordinate threads internally but each pipeline is still single-consumption. |
| Dataset<br>RowDataset | Two-dimensional tabular data structure (interface `Dataset` + final impl `RowDataset`) combining relational-DB and spreadsheet semantics with named, strongly-typed columns and column-oriented storage. Provides projection/slicing (`slice`, `copy`), filtering (`filter`), sorting (`sortBy`), set ops (`union`, `intersect`, `except`), joins (`innerJoin`, `leftJoin`, `rightJoin`, `fullJoin`), grouping/aggregation (`groupBy` with `Collector`), pivoting to `Sheet` (`pivot`), row-to-bean conversion (`toList`, `toEntities`, `toMap`), iteration/streaming (`stream`, rows/columns iterators), and I/O for JSON/XML/CSV (`toJson`, `toCsv`, `Dataset.rows`, factories on `N`/`CsvUtil`). Memory-efficient with lazy evaluation where possible; mutability and thread-safety follow the standard collection model (not synchronized). |
| Sheet | Final two-dimensional table indexed by separate row keys, column keys, and cell values (`Sheet<R, C, V>`) — effectively a 3-axis cross-tab structure. | Multimap<br>ListMultimap<br>SetMultimap | Sealed `Multimap<K, E, V extends Collection<E>>` mapping each key to a collection of values (`put`, `putMany`, `get`, `remove`, `removeOne`, `removeAll`, `flatValues`, `forEach`, `stream`, `entryStream`, `toMap`, `filter`, `inverse`). | BiMap | Final bidirectional 1-to-1 map enforcing uniqueness of both keys and values, with O(1) forward and reverse lookup. | Builder | Fluent builder facade (`Builder<T>`) wrapping a mutable object and exposing it through method-chaining for in-place mutation; `Builder.of(value)` auto-selects an appropriate specialized sub-builder. | Joiner | Final fluent string-joining utility with configurable separator, key-value separator, prefix, suffix, and null handling (`skipNulls`, `useForNull`, `trimBeforeAppend`, `setEmptyValue`). | Difference<br>KeyValueDifference<br>MapDifference<br>BeanDifference | Sealed class for computing differences between two collections, arrays, maps, or beans. `Difference.of(left, right)` returns common / left-only / right-only partitions; collection comparison is occurrence-aware (per-element frequency rather than set semantics). | Tuple<br>Tuple0..Tuple9 | Immutable, fixed-arity heterogeneous container hierarchy (`Tuple<TP>` self-typed CRTP base) implementing `Immutable`. | Pair<br>Triple | Mutable two- and three-element heterogeneous containers. **Pair&lt;L, R&gt;** implements `Map.Entry<L, R>` and `Mutable` — `Pair.of(l, r)`, `left()`/`right()`, `setLeft`/`setRight`, `getKey`/`getValue`/`setValue`, plus `reverse`, `copy`, `forEach`, `accept`, `map`, `filter`, conversion to `Tuple2`/`Optional`. **Triple&lt;L, M, R&gt;** is the analogous mutable three-slot container with `left/middle/right` accessors and mutators, plus `reverse`, `copy`, `accept`, `map`, `filter`, conversion to `Tuple3`.
