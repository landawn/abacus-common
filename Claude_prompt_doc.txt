  ▎ - The method body has a reachable return null (directly, or via a ternary/branch).
  ▎ - It returns a field, parameter, or local that can hold null.
  ▎ - It delegates to / returns the result of another method that can return null (trace one hop; N frequently delegates to CommonUtil).
  ▎ - Default-value overloads count only if the supplied default can itself be null.
  ▎
  ▎ Explicitly out of scope (never null — do not touch):
  ▎ - Methods returning a primitive (int, boolean, etc.).
  ▎ - Methods returning Optional, OptionalInt/OptionalLong/OptionalDouble, Nullable, or other never-null wrappers.
  ▎ - Methods that return an empty array/collection/string instead of null — unless they can also return null.
  ▎ - void methods.
  ▎
  ▎ What to write in @return:
  ▎ - Describe the normal return value and append the null condition, e.g.
  ▎ @return the matched value, or {@code null} if no element matches /
  ▎ @return the parsed date, or {@code null} if {@code source} is {@code null} or empty.
  ▎ - Use {@code null} (not bare "null"), and keep wording consistent across the batch.
  ▎ - If an @return already mentions null, only refine it if the stated condition is wrong or incomplete — don't duplicate or churn correct docs.
  ▎ - If a method has no @return tag at all but returns a value, add one.
  ▎
  ▎ Constraints & verification:
  ▎ - Do not change code, deprecate anything, or reorder members. Javadoc text only.
  ▎ - After each class, confirm it still compiles and is doclint-clean for the changed methods (no new javadoc/-Xdoclint warnings).
  ▎ - Overloaded methods: document each overload on its own (they often differ in null behavior).
  ▎
  ▎ Approach (these files are large — N.java alone is ~42K lines):
  ▎ - Process the classes in parallel batches, but trace each null claim before editing — confirm the actual return null path rather than assuming from the method name.
  ▎ - Produce a short per-class summary: methods updated, and any method where the null condition was non-obvious or where you chose not to change existing docs.
